# 系统端到端使用流程

本文档面向系统使用者，说明从打开系统到拿到竞品分析结果的完整操作流程。

## 1. 打开系统工作台

进入系统首页后，默认会看到竞品分析工作台。页面主要包含任务输入区、运行状态区、报告预览区、报告库、问卷模块、Issue 模块和知识库模块。

## 2. 填写竞品分析需求

在任务输入框中填写你想分析的产品或方向，例如：

```text
帮我分析 AI IDE 领域的主要竞品
```

也可以写得更具体，例如：

```text
帮我分析面向企业研发团队的 AI 编程助手竞品，重点关注定价、IDE 集成、代码安全和团队协作能力
```

## 3. 补充可选参数

如果你已经知道要分析哪些竞品，可以在竞品输入框中填写竞品名称，例如：

```text
Cursor, GitHub Copilot, CodeBuddy
```

如果不填写，系统会自动搜索并发现候选竞品。

你还可以按需要调整搜索数量、搜索后端、质量闭环开关、最大迭代次数、问卷分析文本等参数。一般情况下保持默认即可。

## 4. 启动分析任务

点击“开始分析”后，系统会创建一个新的分析任务。

任务启动后，页面会显示当前任务状态。你可以在运行状态区看到系统正在执行的阶段，例如准备中、搜索竞品、分析中、生成报告、质检闭环、已完成等。

## 5. 等待候选竞品出现

如果系统需要你确认候选竞品，页面会展示自动发现的产品列表。

你可以从列表中选择要继续分析的竞品，也可以手动输入你希望分析的竞品名称。

确认后，系统会继续执行后续分析。

## 6. 查看运行进度

任务运行过程中，你可以查看：

1. 当前执行阶段。
2. 已生成的搜索词。
3. 候选竞品列表。
4. 分析子任务状态。
5. 实时运行日志。

这些信息用于判断系统是否还在正常运行，以及当前分析推进到了哪一步。

## 7. 等待报告生成

系统会自动完成资料搜索、竞品分析、对比整理和报告撰写。

如果开启了质量闭环，系统还会自动检查报告质量，并在发现问题时尝试修复。你不需要手动干预，除非系统提示需要选择竞品或任务失败。

## 8. 查看最终报告

任务完成后，系统会自动加载最新报告。

你可以在报告预览区查看完整 Markdown 报告。报告通常包含：

1. 执行摘要。
2. 竞品概览。
3. 核心功能对比。
4. 定价或商业模式对比。
5. SWOT 分析。
6. 产品机会和风险。
7. 面向产品经理的建议。
8. 信息缺口或待补充项。

## 9. 打开报告库

进入报告库后，可以查看历史任务生成的报告。

报告会按任务分组展示。一个任务下可能包含最终报告、分析总报告、单品报告和质检报告。

点击任意报告文件，可以在侧边预览窗口中查看内容。

## 10. 查看 Issue 明细

如果系统生成了质检结果，可以进入 Issue 页面查看问题列表。

Issue 页面会展示每个任务下发现的问题，例如：

1. 哪些章节信息不足。
2. 哪些表格字段待补充。
3. 哪些结论缺少证据。
4. 哪些建议不够可执行。

你可以根据这些 Issue 判断报告是否还需要人工补充。

## 11. 下载或复制报告内容

在报告预览页中，你可以阅读最终报告，也可以通过下载入口保存报告文件。

如果需要提交作业、整理汇报或继续编辑，可以直接使用最终 Markdown 报告作为基础。

## 12. 使用问卷生成模块

如果你需要补充用户调研，可以进入问卷模块。

使用流程如下：

1. 输入产品描述或分析目标。
2. 设置题目数量。
3. 填写已知竞品，或让系统自动搜索竞品。
4. 点击生成问卷。
5. 查看生成的问卷文件。

生成的问卷可以用于真实调研，也可以继续使用系统的模拟填写功能做快速测试。

## 13. 模拟填写问卷

在问卷模块中选择一份已生成的问卷，然后点击模拟填写。

系统会生成多份模拟回答，并保存为回答文件。这个功能适合用于演示、测试或没有真实用户数据时的快速验证。

## 14. 分析问卷结果

选择问卷文件和回答文件后，点击分析问卷。

系统会生成一份问卷分析报告，通常包括回答统计、用户偏好、痛点总结和产品启发。

这份问卷分析可以作为竞品分析报告的补充材料。

## 15. 构建技能知识库

如果某份报告中有可以复用的分析经验，可以进入知识库模块。

使用流程如下：

1. 选择一份已生成的报告。
2. 点击构建知识库。
3. 等待系统抽取报告中的方法、结构和经验。
4. 查看生成的 skill wiki。

后续可以在知识库聊天框中基于这些沉淀内容提问。

## 16. 复查历史任务

你可以随时回到报告库查看历史任务。

每个任务通常会保留多个文件，包括最终报告、过程报告、质检报告和问卷分析报告。通过任务分组可以快速定位同一次分析产生的全部材料。

## 17. 推荐的完整使用顺序

一次完整使用建议按下面顺序进行：

1. 打开系统工作台。
2. 输入产品或竞品分析需求。
3. 可选填写已知竞品和补充材料。
4. 点击开始分析。
5. 如系统提示，确认候选竞品。
6. 等待系统完成搜索、分析、报告生成和质检。
7. 阅读最终报告。
8. 查看 Issue 页面，确认是否有明显缺口。
9. 如需用户调研，进入问卷模块生成并分析问卷。
10. 如需复用经验，选择报告构建知识库。
11. 在报告库中下载或复查最终材料。
